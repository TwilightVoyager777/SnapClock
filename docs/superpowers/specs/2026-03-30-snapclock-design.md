# SnapClock 产品设计规格文档

**版本：** 1.0
**日期：** 2026-03-30
**作者：** 独立开发者（个人使用项目）
**平台：** iOS + watchOS（Apple Watch Series 7/8/9）

---

## 1. 产品定位与核心价值

### 1.1 核心价值主张

> **"不是在你设定的时间叫醒你，而是在你睡够了之后叫醒你。"**

普通闹钟和倒计时 App 的共同盲区：计时从"按下按钮"开始，而非从"真正入睡"开始。对于入睡时间不固定的人，这意味着：

- 设 30 分钟，实际只睡了 10 分钟（没睡够，被叫醒反而更累）
- 或者担心睡不够，多设 20 分钟缓冲（结果进入深睡被硬拽出来）

SnapClock 解决的是"入睡时间不确定"导致的计时起点偏移问题。

### 1.2 差异化定位

| 维度 | 普通闹钟 | 睡眠 App（Sleep Cycle 等） | SnapClock |
|---|---|---|---|
| 计时起点 | 按下按钮时 | 不关心 | **检测到入睡时** |
| 核心目标 | 在某个时刻叫醒 | 分析睡眠质量 | **睡够指定时长后叫醒** |
| 使用场景 | 全天候 | 夜间完整睡眠 | **小睡（午休/碎片休息）** |
| 需要 Watch | 否 | 可选 | **是（核心依赖）** |

### 1.3 目标用户与场景

**主要用户：** 在固定时间窗口内午休、但入睡时间不稳定的人。

**典型场景：** 午休 30 分钟，躺下后需要 5～20 分钟才真正入睡，希望确保从真正入睡起睡足 30 分钟再被叫醒。

**使用环境：** 家中或办公室，躺在沙发或床上，iPhone 和 Apple Watch 均在身边。

## 2. 平台架构与职责划分

### 2.1 整体架构方案

采用**双引擎分工**架构：iPhone 负责 UI 和数据持久化，Watch 负责会话期间的传感器采集、入睡检测、计时和唤醒。会话期间 Watch 完全自主运行，不依赖 iPhone 连接。

```
┌─────────────────────────────┐     ┌─────────────────────────────┐
│         iPhone App          │     │       Apple Watch App        │
├─────────────────────────────┤     ├─────────────────────────────┤
│ · 设置小睡时长（主界面）      │     │ · 镜像设置界面（快速调整）    │
│ · 历史记录 & 统计            │     │ · 会话状态显示（实时）        │
│ · 会话结束后的数据展示        │◄───►│ · 入睡检测（核心逻辑）        │
│ · 偏好设置（震动强度等）      │     │ · 计时器（检测到入睡后启动）  │
│ · 会话触发（发指令给 Watch）  │     │ · 震动唤醒（Haptic）         │
└─────────────────────────────┘     │ · iPhone 备用通知兜底        │
                                    └─────────────────────────────┘
```

### 2.2 会话期间数据流

```
用户设定时长
     ↓
iPhone 通过 Watch Connectivity（WCSession）发送配置给 Watch
     ↓
Watch 启动 HKWorkoutSession（.other 类型，静默）
  + WKExtendedRuntimeSession（.smartAlarm 类型，专为睡眠场景设计）
     ↓
Watch 持续采集：
  · 心率（HKAnchoredObjectQuery，~5秒/次）
  · 加速度计（CMMotionManager，实时）
     ↓
Watch 本地运行入睡判断算法
     ↓
检测到入睡 → Watch 本地启动倒计时
     ↓
倒计时结束 → Watch 触发 Haptic + Watch 本地通知
     ↓
会话结束 → Watch 将会话数据同步回 iPhone（WCSession transferUserInfo）
     ↓
iPhone 持久化存储（SwiftData）+ 展示会话摘要
```

### 2.3 通知触发策略

- **主闹钟：** Watch 本地触发（Haptic 渐强 + Watch 本地通知）
- **备用兜底：** iPhone 在会话开始时注册一个延迟本地通知，触发时刻 = 会话开始时刻 + 超时保护上限（45分钟）+ 用户设定小睡时长 + 2分钟缓冲。例如：设定 30 分钟小睡，备用通知在 77 分钟后触发。Watch 正常唤醒用户后，立即取消此通知。

### 2.4 关键 API

| API | 用途 | 说明 |
|---|---|---|
| `HKWorkoutSession` | 获取连续高频心率 | `.other` 类型，不显示运动界面，约 5 秒/次心率 |
| `WKExtendedRuntimeSession` | Watch 后台持续运行 | 优先使用 `.smartAlarm` 类型（专为睡眠检测 + 闹钟设计，时长更长）；如不可用，回退 `.mindfulSession`（约 1 小时） |
| `CMMotionManager` | 加速度计采集 | 检测身体静止状态 |
| `WCSession` | iPhone ↔ Watch 通信 | 传递配置指令和会话结果 |
| `UNUserNotificationCenter` | 本地通知 | iPhone 备用闹钟 |
| `SwiftData` | iPhone 端数据持久化 | 存储历史会话记录 |

## 3. 功能设计

### 3.1 MVP 功能清单

#### 必做（第一版）

- **小睡时长设置：** iPhone 和 Watch 双端均可设置
  - 快捷选项：15 / 20 / 25 / 30 / 45 / 60 分钟
  - 自定义输入（5～120 分钟范围）
- **会话启动：** 一键开始，Watch 震动一次确认接收
- **入睡检测：** 基于心率 + 加速度计的复合判断算法（见第 4 节）
- **自适应计时：** 检测到入睡后开始倒计时
- **震动唤醒：** Watch Haptic 渐强模式（3次，间隔 3 秒）+ iPhone 备用通知兜底
- **会话状态显示：** Watch 实时显示"监测中 / 已入睡，剩余 X 分钟"
- **会话摘要：** 结束后显示"你在 X 分钟后入睡，共睡了 Y 分钟"
- **超时保护：** 监测超过 45 分钟未检测到入睡，自动以当前时刻为起点开始计时并通知用户
- **手动确认模式：** 用户可跳过检测，手动标记"现在开始计时"

#### 后续迭代（V2+，不在 MVP 范围内）

- 历史记录 & 趋势图（入睡时长分布、平均睡眠质量）
- Watch Complication（表盘快捷启动）
- 智能时长建议（根据历史数据推荐）
- 渐强唤醒音乐（配合震动）
- 睡眠阶段估计（浅睡 vs 深睡提示）
- Siri / Shortcuts 集成
- 检测灵敏度个性化调整界面

### 3.2 用户使用流程

```
[iPhone 或 Watch]
用户打开 App → 选择小睡时长 → 点击"开始"
        ↓
iPhone 发送配置给 Watch，Watch 震动确认
        ↓
用户放下手机，戴好手表，闭眼休息
        ↓
        ┌─────────────────────────────────┐
        │     Watch 后台静默监测中...      │
        │   心率采集（5秒/次）+ 动作检测   │
        │     滑动窗口算法持续计算         │
        └─────────────────────────────────┘
        ↓
[入睡检测触发]
Watch 表盘切换为"已入睡，30:00 开始倒计时"
        ↓
倒计时结束
        ↓
Watch 渐强震动（×3，间隔 3 秒）
+ iPhone 备用通知（晚 2 分钟触发，如未取消）
        ↓
用户查看摘要："你在 12 分钟后入睡，共睡了 30 分钟"
```

### 3.3 关键界面设计

#### iPhone 端

| 界面 | 核心元素 |
|---|---|
| 主页 | 时长选择轮盘/快捷按钮、大号"开始"按钮、上次记录摘要 |
| 会话中 | 状态文字（监测中/已入睡）、剩余时间、"提前结束"按钮 |
| 会话结束 | 入睡用时、实际睡眠时长、简洁图示 |
| 设置 | 超时保护时长、备用通知开关 |

#### Watch 端

| 界面 | 核心元素 |
|---|---|
| 主页 | 时长 ±5分钟调节、"开始"按钮、"手动计时"选项 |
| 监测中 | "监测中"状态、波形或呼吸动画、取消按钮 |
| 倒计时中 | 大号剩余时间、入睡时刻标记（"12分钟后入睡"）|
| 会话结束 | 入睡用时、实际睡眠时长 |

## 4. 入睡检测逻辑

### 4.1 可用信号分析

| 信号 | 获取方式 | 入睡时的变化 | 可靠性 |
|---|---|---|---|
| 心率（HR） | `HKWorkoutSession` + `HKAnchoredObjectQuery` | 从清醒静息 ~70bpm 降至 ~55bpm | 高，但个体差异大 |
| 心率变异性（HRV） | HealthKit | 入睡后 HRV 上升 | 中，watchOS 采样频率受限 |
| 加速度计 | `CoreMotion` | 身体静止，几乎无动作 | 高，但"静躺未睡"难区分 |
| 手腕抬起 | `WKInterfaceDevice` | 入睡后不会抬手 | 低，仅辅助 |

**结论：单信号均不可靠，必须多信号融合。**

### 4.2 MVP 检测算法

采用**滑动窗口 + 多条件联合触发**：

```
每 5 秒采集一次：当前心率、过去 30 秒加速度计均值

连续 180 秒（3分钟）同时满足以下 3 个条件 → 判定为入睡：

条件1：心率相对降幅
  当前 HR ≤ 基准 HR × 0.88
  （基准 HR = 会话开始时连续 30 秒心率均值）

条件2：心率绝对值
  当前 HR < 65 bpm

条件3：身体静止
  过去 180 秒加速度计三轴标准差 < 0.02g

触发后：锁定入睡时刻，启动用户设定时长的倒计时
中断时：若任一条件不满足超过 30 秒，重置整个 180 秒窗口重新计数；
        若中断 ≤30 秒（如短暂翻身），该段视为满足条件继续累计（每次会话仅允许一次此豁免）
```

### 4.3 误判与漏判控制

| 场景 | 控制策略 |
|---|---|
| 静躺刷手机（未睡） | 心率不充分下降 + 手腕有微动，双条件过滤 |
| 心率天然偏低的用户 | 使用相对降幅（12%）而非绝对阈值 |
| 入睡时短暂翻身 | 30 秒以内的中断不重置窗口（一次豁免） |
| 压根检测不到入睡 | 45 分钟超时保护自动开始计时 + 用户通知 |

### 4.4 体验兜底机制

1. **超时保护：** 监测 45 分钟未检测到入睡 → 自动以当前时刻为起点开始计时，通知"未能检测到明确入睡，已自动开始计时"
2. **手动确认模式：** 用户可随时点击"现在开始"跳过检测，直接进入倒计时

## 5. 系统限制说明

| 限制 | 影响 | 规避方式 |
|---|---|---|
| watchOS 后台默认挂起 | Watch App 无法持续后台运行 | 使用 `WKExtendedRuntimeSession` |
| 心率后台采样频率低（5-10分钟/次） | 检测精度严重不足 | 启动 `HKWorkoutSession` 提升至 ~5秒/次 |
| `WKExtendedRuntimeSession` 时长受类型限制 | 类型选择影响可用时长 | 优先 `.smartAlarm`（睡眠专用，时长更宽松）；MVP 设定上限 60 分钟，开发阶段实测验证 |
| Watch Connectivity 睡眠期间可能断连 | Watch 无法实时回传数据 | Watch 完全本地运行，仅会话结束后同步 |

## 6. 独立开发者开发顺序

### 阶段 0：基础搭建（1-2天）
- 创建 Xcode 多目标项目（iOS + watchOS）
- 配置 HealthKit、CoreMotion 权限（Info.plist + Entitlements）
- 搭建 `WCSession` 基础通信框架
- 验证 iPhone ↔ Watch 双向消息传递

### 阶段 1：传感器验证（3-5天）⭐ 最关键
- Watch 端启动 `HKWorkoutSession`，实时打印心率
- 接入 `CMMotionManager`，打印加速度计数据
- 本人实际躺下测试，采集并记录真实入睡前后数据曲线
- 根据真实数据验证/调整算法阈值

> **此阶段无需正式 UI，用调试界面即可。数据不达预期则优先调整算法，再继续后续开发。**

### 阶段 2：检测与唤醒核心逻辑（3-5天）
- 实现滑动窗口检测算法（Watch 端）
- 加入 `WKExtendedRuntimeSession` 保证后台运行
- 实现检测触发后的本地倒计时
- 实现时间到后的 Haptic 渐强唤醒
- 实现 45 分钟超时保护

### 阶段 3：Watch UI（3-4天）
- 主页：时长选择 + 开始按钮 + 手动模式入口
- 监测中页：状态 + 取消
- 倒计时页：剩余时间
- 会话结束页：摘要

### 阶段 4：iPhone UI + 同步（3-4天）
- 主页：时长选择 + 开始（发指令给 Watch）
- 会话中页：状态镜像
- Watch → iPhone 会话结束数据同步
- iPhone 端会话摘要展示
- iPhone 备用本地通知注册与取消

### 阶段 5：打磨与调优（持续）
- 根据实际使用体验调整检测灵敏度参数
- 历史记录存储（SwiftData）
- Watch Complication 快速启动
- V2 功能迭代

## 7. 技术栈汇总

| 层级 | 技术选型 |
|---|---|
| 语言 | Swift 5.9+ |
| UI 框架 | SwiftUI（iPhone + Watch 统一） |
